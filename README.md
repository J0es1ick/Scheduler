# Scheduler

Telegram-бот автоматически загружает актуальные снимки расписаний:

- ИГХТУ — <https://www.isuct.ru/student/schedule>;
- ИГЭУ — <http://schedule.ispu.ru/>.

Данные сохраняются в PostgreSQL, после чего бот выдаёт расписание выбранной
группы. Для ИГЭУ номер вводится в формате `1-40`, для ИГХТУ — `3/147`.

## Запуск

1. Заполните `.env` по образцу `.env.example`.
2. Запустите PostgreSQL:

   ```powershell
   docker-compose up -d postgres
   ```

3. При необходимости выполните первый импорт отдельно:

   ```powershell
   go run ./cmd/sync
   ```

   Команда без аргументов обновляет оба вуза. Один источник можно запустить
   отдельно: `go run ./cmd/sync ispu-main` или `go run ./cmd/sync isuct-main`.

4. Запустите бота:

   ```powershell
   go run ./cmd/bot
   ```

Миграции применяются автоматически. Фоновый worker запускает проверку сразу при
старте, затем каждые пять минут проверяет, пришло ли время обновить источник.
ИГХТУ и ИГЭУ обновляются раз в час; неуспешный проход повторяется на следующей
проверке. Полный первичный импорт занимает несколько минут.

## Проверка

```powershell
go test ./...
```

Live-тест реального сайта (полный список групп и расписание контрольной группы):

```powershell
$env:ISUCT_INTEGRATION_TEST='1'
go test ./internal/scrapper/isuct -run TestLiveISUCTAdapter -v -count=1
```

Живой тест ИГЭУ:

```powershell
$env:ISPU_INTEGRATION_TEST='1'
go test ./internal/scrapper/ispu -run TestLiveISPUAdapter -v -count=1
```

Сайт отдаёт не обычную HTML-форму: autocomplete работает только через POST и
ограничивает ответ 15 группами, а расписание приходит в Drupal AJAX JSON.
Адаптер обходит ограничение autocomplete, повторяет временно повреждённые ответы
и публикует данные группы в БД транзакционно.

ИГЭУ использует ASP.NET WebForms. Его адаптер обходит все опубликованные виды
расписания, факультеты, курсы, группы и подгруппы, а экзамены, зачёты и сессии
сохраняет по точным календарным датам.
